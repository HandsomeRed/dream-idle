# 2026-03-12 学习日志 - 游戏开发

## 📚 学习内容 - 放置挂机游戏设计

### 核心概念

**Idle Game（放置游戏）定义：**
- 也称增量游戏（incremental games）或点击游戏（clickers）
- 将「现实时间」作为游戏要素
- 玩家无需持续操作，游戏自动推进
- 特点：高留存率（18% vs 10.5%）、长会话时长（平均 8 分钟）

### 放置游戏核心要素

1. **反馈循环** - 立即反馈激励玩家继续投入
2. **增量进展** - 不断增加收益速率，解锁新内容
3. **简单控制** - 操作简单，任何玩家都能上手
4. **自动化收益** - 离线也能获得进度，促使定期返回

### 设计要点

**放置 ≠ 无核心：**
- 放置本身不是核心，需要结合其他玩法
- 例如：最强蜗牛 = 战力值比拼 + 放置
- 剑与远征 = 卡牌策略战斗 + 放置

**玩家习惯养成：**
- 设计离线收益机制
- 设置定时奖励
- 创建长期目标

### 技术参考

**开源项目：**
- [vue-idle-game](https://github.com/Couy69/vue-idle-game) - Vue3 实现的刷装备放置游戏
- 支持 Docker 部署、手机界面、存档导入导出

**开发工具：**
- 前端：React/Vue + TypeScript
- 引擎：Unity（适合复杂放置游戏）
- 简单 Web：纯 HTML/CSS/JS 即可

### 梦幻西游题材改编思路

**原作核心：**
- 回合制战斗
- 宠物养成
- 经济系统
- 社交玩法

**放置化改造：**
1. 战斗 → 自动回合战斗 + 离线挂机
2. 任务 → 自动寻路 + 定时领取
3. 经济 → 自动产出 + 离线收益
4. 社交 → 异步交互 + 放置排行榜
