## 📖 核心概念

**只总结有通用价值的 Bug 修复**，避免知识库被琐碎问题淹没。

---

## 🔧 判断标准

### ❌ 不总结（琐碎 Bug）

| 类型 | 示例 | 原因 |
|:---|:---|:---|
| 语法错误 | 少了括号、拼写错误 | 太琐碎，无通用价值 |
| 命名错误 | 变量名写错、函数名错误 | 粗心导致，非系统性问题 |
| 配置错误 | 路径写错、端口错误 | 一次性问题 |
| 临时修复 | 注释掉某行、回滚代码 | 非正式解决方案 |

### ✅ 总结（典型案例）

| 类型 | 示例 | 通用价值 |
|:---|:---|:---|
| **网络超时** | API 请求超时处理 | 可复用的错误处理模式 |
| **内存泄漏** | 未释放资源、循环引用 | 资源管理最佳实践 |
| **并发问题** | 竞态条件、死锁 | 并发编程模式 |
| **异步问题** | Promise 未处理、回调地狱 | 异步编程最佳实践 |
| **类型转换** | 隐式转换导致的 bug | 类型安全实践 |
| **编码问题** | 中文乱码、字符集问题 | 编码规范 |
| **性能问题** | 慢查询、内存优化 | 性能优化模式 |

## 📊 实际案例

### 案例 1：网络超时处理 ✅ 值得总结

**问题：** AKShare API 请求偶尔超时

**根本原因：** 未设置超时时间，网络波动时程序卡死

**解决方案：**
```python
# 添加超时重试机制
import requests
from requests.adapters import HTTPAdapter
from urllib3.util.retry import Retry

session = requests.Session()
retry = Retry(total=3, backoff_factor=0.3)
adapter = HTTPAdapter(max_retries=retry)
session.mount('http://', adapter)
```

**通用价值：** 所有网络请求都可以用这个模式

**决策：** ✅ 总结到知识库

### 案例 3：异步竞态条件 ✅ 值得总结

**问题：** 多个异步操作顺序不确定导致数据错误

**根本原因：** 未正确处理 Promise 依赖关系

**解决方案：**
```typescript
// 使用 async/await 保证顺序
async function fetchData() {
  const user = await getUser();
  const orders = await getOrders(user.id);
  return { user, orders };
}
```

**通用价值：** 所有异步编程场景

**决策：** ✅ 总结到知识库

---

### 案例 4：配置文件路径错误 ❌ 不总结

**问题：** `config.json` 路径写成了绝对路径

**根本原因：** 开发环境和生产环境路径不一致

**解决方案：** 改为相对路径

**通用价值：** 低（项目特定）

**决策：** ❌ 不总结（除非是多环境配置的最佳实践）

## ⚠️ 注意事项

1. **一事一总结** - 同类 Bug 只总结一次
2. **提炼模式** - 从具体问题抽象出通用模式
3. **代码示例** - 必须有可复用的代码
4. **预防措施** - 告诉别人如何避免

---

## 🎯 知识库质量优先

**原则：** 宁可少总结，不要总结垃圾

**目标：** 每条知识都有人愿意回头看

**过滤机制：**
- 琐碎 Bug → 跳过
- 一次性问题 → 跳过
- 有通用价值 → 总结
- 可复用模式 → 重点总结
