## 📖 核心概念

**完全自动的知识沉淀机制**：AI 助手在完成任务后，自动判断是否值得总结，并自动保存到知识库。

**核心原则：** AI 自己判断什么值得总结，不需要人工干预。

---

## 🔧 实现逻辑

### 1. 触发时机

```
任务完成 → 更新状态 (completedAt)
   ↓
心跳检查 (每 5 分钟)
   ↓
检测新完成的任务 (10 分钟内)
   ↓
判断是否值得总结
   ↓
生成知识卡片 → 保存到知识库
```

### 2. 判断标准

#### ✅ 值得总结的

1. **新功能开发** - 如"数据获取模块"、"均线策略"
2. **新技术学习** - 如"AKShare 使用方法"
3. **架构设计** - 如"回测引擎架构"
4. **最佳实践** - 如"测试驱动开发流程"
5. **可复用模式** - 如"数值设计公式"
6. **重要 Bug 解决** - 如"网络超时处理"

#### ❌ 不值得总结的

1. **琐碎的 bug** - 如"拼写错误"、"少了一个括号"
2. **配置调整** - 如"修改端口号"
3. **临时修复** - 如"注释掉某行代码"
4. **重复性工作** - 如"复制粘贴代码"

### 3. 关键词过滤

```javascript
// 不值得总结的关键词
const skipKeywords = [
  '修复', 'bug', 'typo', '拼写',
  '配置', '端口', '颜色',
  '临时', '注释', '重命名'
];

// 值得总结的关键词
const valuableKeywords = [
  '实现', '开发', '系统', '模块',
  '策略', '引擎', '架构', '设计',
  '学习', '方法', '实践', '理论',
  '公式', '算法', '模型', '框架'
];
```

## 📊 工作流程

```javascript
// capture-knowledge.js 核心逻辑

function main() {
  // 1. 读取学习状态
  const state = readLearningState();
  
  // 2. 获取当前时间段
  const timeSlot = getCurrentTimeSlot();
  
  // 3. 检查新完成的任务
  for (const task of progress) {
    if (task.completed && !task.knowledgeCaptured) {
      
      // 4. 判断是否值得总结
      if (!isWorthCapturing(task)) {
        console.log('⚠️ 琐碎内容，跳过');
        continue;
      }
      
      // 5. 生成知识卡片
      const content = generateKnowledgeCard(task);
      
      // 6. 保存到知识库
      const filepath = saveToKnowledgeBase(content);
      
      // 7. 标记为已总结
      task.knowledgeCaptured = true;
    }
  }
}
```

## ⚠️ 注意事项

1. **避免重复** - 已总结的任务不再重复总结
2. **时间窗口** - 只总结 10 分钟内完成的任务
3. **质量优先** - 宁可少总结，不要总结垃圾
4. **灵活调整** - 根据实际效果调整判断标准
