- ❌ 外网访问超时

---

## 🔎 排查步骤

### 步骤 1: 确认安全组绑定

**请登录腾讯云控制台检查：**

1. 访问：https://console.cloud.tencent.com/cvm/instance/index
2. 找到你的实例（IP: 49.232.215.84）
3. 点击实例 ID 进入详情页
4. 点击 **"安全组"** 标签
5. **截图发给我：** 绑定了几个安全组？每个的名字/ID

**关键问题：**
- 如果绑定了多个安全组，每个都要添加 4001 规则
- 常见情况：默认安全组 + 自定义安全组

---

### 步骤 2: 检查网络 ACL

**网络 ACL 是子网级别的防火墙，独立于安全组**

**检查路径：**
1. 腾讯云控制台 → 私有网络 VPC
2. 找到你的 VPC
3. 点击子网 ID
4. 查看是否绑定了网络 ACL
5. 如果有，检查入站规则

**需要确认：**
- 是否绑定了网络 ACL？
- 入站规则是否有允许 4001 端口？
- 是否有拒绝全部的规则？

---

### 步骤 3: 检查云服务器安全策略

有些云服务器有额外的安全层：

**检查命令：**
```bash
# 查看是否有云安全中心
ps aux | grep -i "security\|safeguard"

# 查看是否有主机安全组件
rpm -qa | grep -i "yunjingtang\|host\|security" 2>/dev/null
dpkg -l | grep -i "security" 2>/dev/null
```

---

### 步骤 4: 测试其他已开放的端口

**确认你的哪些端口能访问：**

| 端口 | 服务 | 状态 |
|:---|:---|:---|
| 22 | SSH | ✅ 能访问 |
| 3000 | 梦幻放置 | ✅ 能访问 |
| 4001 | Slay the Web | ❌ 不能访问 |

**请确认：**
- 3000 端口现在能访问吗？
- 还有其他开放的端口吗？

---

### 步骤 5: 对比配置

**既然 3000 能访问，4001 不能，说明：**
- 安全组可能有 3000 但没有 4001
- 或者网络 ACL 限制了 4001

**请检查安全组规则列表：**
- 是否有 TCP:3000 规则？
- TCP:4001 规则是否真的存在？
- 两条规则的优先级是否一样？

## 🔧 可能的解决方案

### 方案 A: 添加缺失的规则

如果发现安全组或网络 ACL 缺少 4001 规则，添加即可。

### 方案 B: 删除有问题的安全组

如果实例绑定了多个安全组，可以尝试：
1. 只保留一个配置正确的安全组
2. 删除其他安全组绑定

### 方案 C: 联系腾讯云客服

如果以上都正常，可能是腾讯云网关层面的问题，需要提工单。
