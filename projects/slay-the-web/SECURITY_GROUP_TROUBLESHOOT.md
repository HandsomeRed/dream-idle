# 🔍 安全组配置了但端口还是关闭？

## 问题诊断

**现象：** 安全组规则已添加，但端口测试显示"关闭"  
**服务状态：** ✅ 正常监听 `0.0.0.0:4001`

---

## 🔎 可能原因及解决方案

### 原因 1: 安全组未绑定到实例 ⭐⭐⭐⭐⭐

**最常见原因！** 添加了规则但安全组没有绑定到云服务器。

**检查步骤：**
1. 登录腾讯云控制台
2. 进入 **云服务器 (CVM)**
3. 找到你的实例 (49.232.215.84)
4. 点击实例 ID 进入详情页
5. 找到 **"安全组"** 标签
6. **确认：** 列表中是否有你刚才配置的安全组？

**如果没有绑定：**
1. 点击 **"绑定安全组"**
2. 选择你配置了 4001 规则的安全组
3. 点击确定

---

### 原因 2: 有多个安全组 ⭐⭐⭐⭐

如果实例绑定了多个安全组，规则会**叠加**，只要有一个安全组阻止就访问不了。

**检查步骤：**
1. 在实例的安全组标签页
2. 查看绑定了几个安全组
3. 如果超过 1 个，**每个**都要添加 4001 规则

**解决方法：**
- 方案 A：给每个安全组都添加 4001 规则
- 方案 B：只保留一个安全组，删除其他绑定

---

### 原因 3: 规则优先级问题 ⭐⭐⭐

如果有"拒绝全部"的规则优先级更高，会覆盖你的允许规则。

**检查步骤：**
1. 进入安全组规则配置
2. 查看入站规则列表
3. 确认"允许 TCP:4001"的优先级（数字越小优先级越高）
4. 确保它在"拒绝全部"规则**上面**

**解决方法：**
- 调整优先级，把 4001 规则移到最上面
- 或者删除"拒绝全部"规则

---

### 原因 4: 规则未生效 ⭐⭐

安全组规则通常需要 1-5 分钟生效。

**解决方法：**
- 等待 5 分钟
- 期间可以刷新控制台确认规则还在
- 然后用在线工具测试

---

### 原因 5: 配置了"出站规则"而非"入站规则" ⭐⭐

**常见错误！** 需要的是**入站规则**（别人访问你），不是出站规则（你访问别人）。

**检查步骤：**
1. 进入安全组配置
2. 确认是在 **"入站规则"** 标签页
3. 不是"出站规则"

## 🎯 临时解决方案

如果实在搞不定安全组，可以用**已开放的端口**：

### 方案 A: 用 3000 端口（梦幻放置的端口）

**前提：** 梦幻放置现在没在运行

```bash
# 停止当前服务
pkill -f "vite"
pkill -f "python server.py"

# 修改前端端口为 3000
cd /root/.openclaw/workspace/projects/slay-the-web/frontend
# 编辑 vite.config.ts，把 port: 4001 改成 port: 3000

# 启动
npm run dev -- --host 0.0.0.0 --port 3000 &
```

然后访问：http://49.232.215.84:3000

### 方案 B: 用 Nginx 反向代理

如果你有 Nginx，可以配置反向代理：

```nginx
location /slay-the-web/ {
    proxy_pass http://localhost:4001/;
    proxy_http_version 1.1;
    proxy_set_header Upgrade $http_upgrade;
    proxy_set_header Connection "upgrade";
}
```

然后访问：http://49.232.215.84/slay-the-web/
