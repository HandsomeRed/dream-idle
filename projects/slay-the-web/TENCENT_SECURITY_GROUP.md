# 🔧 腾讯云安全组配置指南

## 🎯 问题症状

**错误信息：** "49.232.215.84 响应时间太长"  
**原因：** 连接超时 = 安全组未开放端口

---

## ✅ 正确的安全组配置步骤

### 步骤 1: 登录腾讯云控制台

访问：https://console.cloud.tencent.com/

### 步骤 2: 找到你的云服务器

1. 点击左侧 **"云服务器 (CVM)"**
2. 找到你的实例（IP: 49.232.215.84）
3. 点击实例 ID 进入详情页

### 步骤 3: 查看安全组

1. 在实例详情页，找到 **"安全组"** 标签
2. 点击安全组 ID（通常是一个数字，如 sg-xxxxxxxx）

### 步骤 4: 添加入站规则

1. 点击 **"修改规则"** 或 **"添加规则"**
2. 选择 **"入站规则"**（非常重要！不是出站）
3. 点击 **"添加规则"**
4. 填写：

```
类型：自定义
协议端口：TCP:4001
来源：0.0.0.0/0
策略：允许
优先级：1
描述：Slay the Web 游戏
```

5. 点击 **"保存"**

### 步骤 5: 确认规则生效

1. 回到安全组规则列表
2. 确认能看到刚才添加的规则
3. 规则状态应该是"生效中"

---

## 📸 配置示例

### 正确的入站规则列表：

| 类型 | 协议端口 | 来源 | 策略 | 状态 |
|:---|:---|:---|:---|:---|
| SSH | TCP:22 | 0.0.0.0/0 | 允许 | ✅ |
| HTTP | TCP:80 | 0.0.0.0/0 | 允许 | ✅ |
| 自定义 | TCP:3000 | 0.0.0.0/0 | 允许 | ✅ |
| **自定义** | **TCP:4001** | **0.0.0.0/0** | **允许** | ✅ |

### 错误配置示例 ❌

| 错误 | 说明 |
|:---|:---|
| 配成了"出站规则" | 需要的是"入站规则" |
| 来源不是 0.0.0.0/0 | 限制太严格，外网访问不了 |
| 协议选错了 | 必须是 TCP |
| 端口写错了 | 确认是 4001 不是 4000 或其他 |

## ⏱️ 等待生效

**安全组规则通常在 1-2 分钟内生效**

等待期间可以：
1. 刷新腾讯云控制台确认规则在列表中
2. 用手机 4G 网络测试（关闭 WiFi）
3. 访问在线端口测试工具

## 🔍 常见问题

### Q1: 找不到安全组在哪里？

**路径：** 云服务器 → 实例详情 → 安全组标签

### Q2: 添加了规则还是打不开？

1. 确认是"入站规则"不是"出站"
2. 确认协议是 TCP
3. 确认来源是 0.0.0.0/0
4. 等待 2 分钟让规则生效
5. 用在线工具测试端口

### Q3: 有多个安全组怎么办？

- 每个安全组都要添加规则
- 或者只保留一个安全组，把规则都加进去

### Q4: 规则优先级有影响吗？

- 有！数字越小优先级越高
- 确保允许规则的优先级高于拒绝规则
- 建议设置为 1
