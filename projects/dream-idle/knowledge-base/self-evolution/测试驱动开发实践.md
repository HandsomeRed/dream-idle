## 📖 核心概念

**测试驱动开发 (Test-Driven Development, TDD)** 是一种先写测试、再写实现的开发方法。

**核心流程：** 红 → 绿 → 重构

---

## 🔧 TDD 流程

### 1. 红 (Red) - 写一个失败的测试

```python
# 1. 先写测试
def test_calculate_ma():
    df = pd.DataFrame({'收盘': [10, 11, 12, 13, 14, 15]})
    result = calculate_ma(df, periods=[3])
    
    # 此时期望 MA3 列存在
    assert 'MA3' in result.columns
    # 第 3 个值应该是 (10+11+12)/3 = 11
    assert result['MA3'].iloc[2] == 11.0
```

**运行测试：** ❌ 失败（因为还没实现）

---

### 2. 绿 (Green) - 快速实现让测试通过

```python
# 2. 快速实现
def calculate_ma(df: pd.DataFrame, periods: list) -> pd.DataFrame:
    df = df.copy()
    for period in periods:
        df[f'MA{period}'] = df['收盘'].rolling(window=period).mean()
    return df
```

**运行测试：** ✅ 通过

---

### 3. 重构 (Refactor) - 优化代码

```python
# 3. 重构优化
def calculate_ma(df: pd.DataFrame, periods: list = [5, 10, 20]) -> pd.DataFrame:
    """计算移动平均线"""
    df = df.copy()
    for period in periods:
        df[f'MA{period}'] = df['收盘'].rolling(window=period).mean()
    return df
```

**运行测试：** ✅ 依然通过

## ⚠️ 注意事项

1. **测试粒度** - 每个测试只测一个功能点
2. **测试命名** - 清晰表达测试意图
3. **边界条件** - 测试边界值和异常情况
4. **避免过度测试** - 不测试 getter/setter 等简单方法
5. **测试维护** - 重构时同步更新测试

---

## 🎯 TDD 优势

1. **设计驱动** - 先思考接口，再实现
2. **质量保障** - 测试覆盖率高
3. **重构信心** - 有测试保护，重构更安全
4. **文档作用** - 测试即文档，展示如何使用

## 📝 测试模板

```python
#!/usr/bin/env python3
# -*- coding: utf-8 -*-
"""
模块单元测试
"""

import pytest
import pandas as pd
from your_module import your_function


class TestYourFunction:
    """测试类"""
    
    def test_normal_case(self):
        """测试正常情况"""
        result = your_function(input_data)
        assert result == expected_output
    
    def test_edge_case(self):
        """测试边界情况"""
        result = your_function(edge_input)
        assert result == edge_expected
    
    def test_error_case(self):
        """测试异常情况"""
        with pytest.raises(ValueError):
            your_function(invalid_input)


if __name__ == "__main__":
    pytest.main([__file__, "-v"])
```
